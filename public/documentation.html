API Documentation

URL Business Logic HTTP Method Query Parameters Data Type Request Body Data Format Response Body Data Format
/movies Returns a list of ALL movies to the user GET None A JSON object holding data about all the movies.
/movies/:title Returns data about a single movie GET title:String None Return JSON object containing data (title, year, description, genre, director, imageURL) about a single movie.

If the movie title is not found, a text error message "{title} not found" is returned.

Example: 
{ 
"title":"Avatar", 
"year":"2009", 
"description":"", 
"genre":"Action, Adventure,Fantasy", 
"director":"James Cameron", 
"imageURL":""
}
/genre/:genrename Get data about a genre by name GET genrename:String None Returns description about that genre and a JSON array including movies of that genre.

If the genre is not found, a text error message "Genre {genrename} not found" is returned.

Example:                
{
  "name": "Action",
  description": "High-energy films packed with intense fight scenes, fast-paced choreography, and
  adrenaline-pumping sequences that showcase the power and skill of martial artists in combat.",
  "movies": [
      {
          "title": "Enter the Dragon",
          ...
      },
      {
          "title": "Drunken Master",
          ...
      }      
  ]
}
/directors/{dirname} Get data about a director GET dirname:String None Returns a JSON object containing data(bio, birth year, death year,movies) about given directors name.

If the director is not found, a text error message "Director name, {dirname} not found" is returned.

Example:
{
  "name": "Stephen Chow",
  "bio": "Stephen Chow is a Hong Kong filmmaker, actor, and comedian, 
  known for his unique blend of slapstick humor, visual effects, and martial arts.",
  "birthYear": 1962,
  "deathYear": null,
  "movies": [
      {
          "title": "Kung Fu Hustle",
          "releaseYear": 2004,
          "genre": "Action",          
      },
      {
          "title": "Shaolin Soccer",
          "releaseYear": 2001,
          "genre": "Action",          
      }
  ]
}
/users/adduser Register a new user POST None A JSON object holding data about user to add ,structured like: 
{
  "username":"moviebuff",
  "email":"james@example.com",
  "password":"xxxxxx",
  "secret-question":"In which city were you born?",
  "secret-answer":"Atlanta"
}
A text message indicating that new user was added. Response also include a response JSON object with a new unique ID: 
{
  "ID":4,
  "username":"moviebuff",
  "email":"james@example.com",
  "password":"xxxxxx",
  "secret-question":"In which city were you born?",
  "secret-answer":"Atlanta"
}
/users/{username} Allow users to update user info PUT username:String A JSON object holding updated user info,structured like: 
{
  "username":"moviebuff",
  "email":"james@example.com",
  "new_secretanswer":"Atlanta,GA,USA", 
}
A JSON object holding data about the user that was updated.

If the user is not found, a text error message "{username} not found" is returned.

Example:                
{ 
  "username":"moviebuff", 
  "secret-answer":"Atlanta,GA,USA"
}
/users/{username}/favorites Allow users to add a movie info to favorites list POST None A JSON object holding the movie info to be added to favorites list,structured like: 
Example:
{
  "Title": "The Avengers",
  "Year": "2012",
  "Rated": "PG-13",
  "Genre": "Action, Sci-Fi, Thriller",
  "Director": "Joss Whedon",
  "Images": ""  
}
A text message indicating whether the movie info was added successfully.
/users/{username}/favorites/{movieid} Remove a movie from favorites list DELETE username:String, movieid:Number None A text message indicating whether the movie info was added successfully.
/users/{username}/favorites Allow users to add a movie info to favorites list POST None A JSON object holding the movie info to be added to favorites list,structured like: 
Example:
{
  "Title": "The Avengers",
  "Year": "2012",
  "Rated": "PG-13",
  "Genre": "Action, Sci-Fi, Thriller",
  "Director": "Joss Whedon",
  "Images": ""  
}
A text message indicating whether the movie info was added successfully.
/users/{username} Remove/deregister an user DELETE username:String None A text message indicating that user email has been removed.

If the user does not exist, a text error message "{username} not found" is returned.
